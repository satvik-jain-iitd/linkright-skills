# Eval, Changelog Discipline

The Knowledge Curator runs this before any knowledge edit ships. Every item must pass.

- [ ] Every edit has evidence behind it, no hunch changes
- [ ] Every edit is logged with a date and a reason
- [ ] No owner specific fact was baked into a generic engine file
- [ ] The changelog is appended, never overwritten, so changes can be rolled back
- [ ] Instance files hold owner specifics, engine files stay generic

Pass only if every box is checked.
