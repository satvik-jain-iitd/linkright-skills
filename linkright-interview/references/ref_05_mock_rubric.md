# ref_05 — Mock Interview Scoring Rubric

## Scoring Scale

**1** — Poor: answer is vague, shows no evidence, or misses the point of the question
**2** — Below bar: some substance but lacks structure, specifics, or outcome
**3** — Meets bar: answer is clear, structured, has evidence — interviewer would pass
**4** — Strong: answer is crisp, specific, well-structured, shows real PM thinking
**5** — Exceptional: interviewer would flag this candidate positively to team

Score each dimension independently. Don't average early — review pattern across dimensions.

---

## Dimensions

### 1. Structure (1-5)

Does the answer have a clear shape the interviewer can follow?

| Score | What it looks like |
|---|---|
| 1 | Stream-of-consciousness, jumps around |
| 2 | Some structure but breaks down midway |
| 3 | Clear start/middle/end, STAR or equivalent |
| 4 | Structure is tight, transitions are explicit |
| 5 | Structure feels invisible — natural but always traceable |

### 2. Specificity (1-5)

Does the answer have real, concrete details?

| Score | What it looks like |
|---|---|
| 1 | All generalities: "I always involve stakeholders" |
| 2 | One specific but rest are vague |
| 3 | Company name, team, rough metric — enough to verify |
| 4 | Metric, timeframe, who was involved, what changed |
| 5 | Specific enough that interviewer could ask 3 follow-up questions and get real answers |

### 3. PM Signal (1-5)

Does the answer demonstrate core PM thinking: user insight, tradeoffs, prioritization, outcome ownership?

| Score | What it looks like |
|---|---|
| 1 | No PM framing — could be any role |
| 2 | Mentions product but doesn't show PM-specific judgment |
| 3 | Shows one PM dimension clearly (e.g., user insight or tradeoff reasoning) |
| 4 | Shows 2+ PM dimensions, owns the outcome |
| 5 | Interviewer learns something about how this person thinks as a PM |

### 4. Outcome Clarity (1-5)

Is the result of the story stated clearly and owned?

| Score | What it looks like |
|---|---|
| 1 | No outcome mentioned |
| 2 | Outcome vague: "it went well", "stakeholders were happy" |
| 3 | Outcome stated but not quantified |
| 4 | Outcome with metric, clear ownership |
| 5 | Outcome + what you learned + what you'd do differently |

### 5. Delivery (1-5)

Was the answer communicated well — pacing, length, confidence?

| Score | What it looks like |
|---|---|
| 1 | Too long, too short, uncertain, or trails off |
| 2 | One of the above issues present |
| 3 | Clear pacing, answer ends at the right place |
| 4 | Confident, well-paced, minimal filler ("um", "like", "you know") |
| 5 | Interview-ready — would make a strong impression |

## Per-Question Scorecard Template

```
Question: [question text]

Structure:        [1-5] — [1-line note]
Specificity:      [1-5] — [1-line note]
PM Signal:        [1-5] — [1-line note]
Outcome Clarity:  [1-5] — [1-line note]
Delivery:         [1-5] — [1-line note]

Total: [sum] / 25

WHAT WORKED:
  [1-2 specific things done well — be specific]

WHAT TO IMPROVE:
  [1-2 specific, actionable improvements — never vague]

REVISED ANSWER HOOK:
  [1-2 sentences showing what a stronger version would start with]
```

## Session Debrief Format

Run after a full mock session (3-5 questions minimum).

```
MOCK SESSION DEBRIEF — [date]
Questions covered: [N]
Duration: [X minutes]

SCORE SUMMARY:
  Structure avg:        [X.X]
  Specificity avg:      [X.X]
  PM Signal avg:        [X.X]
  Outcome Clarity avg:  [X.X]
  Delivery avg:         [X.X]
  Overall avg:          [X.X / 5]

PATTERN — STRENGTHS:
  [What consistently scored 4+ across answers]

PATTERN — GAPS:
  [What consistently scored ≤2 — this is the practice target]

TOP 1 PRIORITY BEFORE NEXT MOCK:
  [Single most important thing to fix — one answer, one behavior]

QUESTIONS TO REDO:
  [List of Q# where any dimension scored ≤2]
```

---

## Pass Bar by Round Type

| Interview type | Pass bar |
|---|---|
| Recruiter screen | All dimensions ≥ 3, Delivery ≥ 4 |
| Hiring manager screen | PM Signal ≥ 4, Structure ≥ 3, Specificity ≥ 4 |
| Behavioral panel | All dimensions ≥ 3.5 avg |
| Case / product sense | PM Signal ≥ 4, Structure ≥ 4 |
| Final / culture | Delivery ≥ 4, PM Signal ≥ 3.5 |

---

## Common Failure Modes + Fixes

| Failure mode | Symptom | Fix |
|---|---|---|
| Story too broad | Mentions 3 projects in one answer | Pick one, go deep |
| No numbers | "improved metrics significantly" | Find a real number, even rough |
| Passive ownership | "the team decided", "we worked on" | Use "I" — own your contribution |
| Answer doesn't end | Trails off into additional context | End with the result, then stop |
| Weak opener | "So... um... yeah, so basically..." | Rehearse first sentence until it's confident |
| Forgets the question | Answers a different question | Restate the question before answering |
